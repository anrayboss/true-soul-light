# Antigravity IDE 高效使用指南

本指南專為你的核心需求設計：**將影音轉為文字，並進一步轉化為投影片、企劃書與網頁**。

---

## 0. Antigravity IDE 介面區塊介紹

第一次使用時，請先熟悉以下視窗區塊，這能幫助你流暢地進行操作：

* **1. 中央編輯區 (Editor)**
  * 用來檢視與直接修改檔案內容（如 Markdown 文件、HTML、CSS 或 Python 程式碼）。可以直接在這裡打字，也可以選取特定段落後按快捷鍵呼叫 AI。
* **2. 左側 AI 助手視窗 (AI Chat Panel)**
  * **你與 AI（我）互動的核心區域**。在此輸入指令、使用斜線指令（如 `/goal`）、提出疑問，或是直接將檔案從檔案總管拖曳到對話框中，指派 AI 處理。
* **3. 右側側邊欄 (Sidebar)**
  * **檔案總管 (Explorer)**：顯示專案的所有資料夾與檔案，方便你快速切換與管理。
  * **版本控制 (Git)**：顯示檔案修改狀態，讓你輕鬆 commit 存檔或還原。
  * **延伸模組 (Extensions/外掛)**：安裝或管理 IDE 的各類擴充功能，以支援更多程式語言與工具。

## 1. 與 AI (我) 對話的高效眉角與訣竅

為了讓我更精確、省時地幫你工作，請掌握以下幾點：

* **1. 給我精確的檔案連結**
  不要只說「幫我改剛剛那個檔案」。請從右側清單中找到要修改的檔案，點擊並直接拉動到和我的對話框內給我。
  * 這樣我可以直接精確定位，不會找錯檔案。
* **2. 善用互動式命令**
  * **/goal**：適合丟給我不需看管的長任務（例如：一次轉錄 3 個長影音並輸出 3 份簡報與 3 個網頁）。我會進入自動化迴圈，直到完全達成才停。
  * **/grill-me**：當你想寫一份企劃書但沒頭緒時，輸入 `/grill-me`，我會主動向你提問，抽絲剝繭幫你把需求問清楚再開始動手。
* **3. 實作計畫 (Implementation Plan)**
  * 當你要進行比較複雜的網頁開發或架構調整時，我會先在工作區建立一個 `implementation_plan.md`。請花 10 秒鐘看一下我規劃的修改邏輯。你輸入「OK/批准」後，我才會開始寫程式碼，這樣可以避免我做無用功。
* **4. 你的專屬規則已載入**
  * 我會直接給你**實際的程式碼、指令或修改後的內容**，絕不講「你可以這樣做...」等高空廢話。
  * 我會自動使用**繁體中文**與你對話。
  * 若涉及金額，預設會以 **NTD** 呈現，次要為 **USD**。
  * 我會絕對尊重你原有的排版、Prettier 設定以及既有的專有名詞譯名。
* **5. 善用反白選取、直接指定要請我修改的內容**
  * 任何檔案中若有需要修改的部分，可以直接反白，再按自動跳出的「Chat」按鈕，或是鍵盤上的 ctrl+L，將要修改的部分加入和我的對話框中。
  * 這樣我就會直接針對你指定的部分，按照你給予的指令做編輯。

## 2. 核心工作流：從影音到多種產出

你不需要手動執行繁瑣的轉換步驟，直接使用 IDE 內建的**斜線指令**（Workflow）即可自動化完成。

### 🎥 步驟一：影音轉錄成逐字稿與大綱
當你手上有影片或音訊檔時，請使用：
* **斜線指令**：`/轉錄成逐字稿 字幕和大綱`
* **操作方式**：在對話框輸入 `/轉錄成逐字稿 字幕和大綱`，並附上你的檔案路徑。例如：
  > 「請幫我轉錄這個影片：`C:\Users\ASUS\Desktop\lecture.mp4`」
* **產出**：AI 會在背景呼叫轉錄工具，為你產出包含時間軸的逐字稿、字幕檔 (.srt) 以及結構化大綱的 Markdown 檔案。

### 📄 步驟二：文字轉 Word 企劃書/講稿
如果你需要將整理好的 Markdown 講稿轉為正式的 Word 檔傳給客戶：
* **斜線指令**：`/markdown 轉存 word 檔`
* **操作方式**：
  > 「請幫我把 [行銷界九陽神功_完整講稿_最終版.md](file:///c:/Users/ASUS/Desktop/真靈光所有資料總整理/行銷課程整理/行銷界九陽神功_完整講稿_最終版.md) 轉存為 Word 檔。」
* **產出**：AI 會直接在相同目錄下生成對應的 `.docx` 檔案。

### 📊 步驟三：文字轉投影片 (PPTX)
將整理好的大綱一鍵轉為簡報：
* **斜線指令**：
  * `/製作亮色簡報` (簡約白底，適合正式商務報告)
  * `/製作暗色簡報` (科技感黑底，適合提案、演講)
* **操作方式**：
  > 「請使用 `/製作亮色簡報`，幫我把 [行銷界九陽神功_完整講稿_最終版.md](file:///c:/Users/ASUS/Desktop/真靈光所有資料總整理/行銷課程整理/行銷界九陽神功_完整講稿_最終版.md) 轉成 PPTX 簡報。」
* **產出**：自動生成排版完成、包含結構的 `.pptx` 簡報檔案。

### 🌐 步驟四：文字轉精美網頁 (HTML/CSS)
如果你需要把講稿做成行銷落地頁 (Landing Page) 或宣傳網頁：
* **操作方式**：直接對 AI 下達明確的網頁規格需求：
  > 「請幫我把這份講稿改寫為一個精美的單頁 RWD 行銷網頁。使用 HTML 和 Vanilla CSS，配色要金黃色與深色搭配的 premium 質感，並且加入平滑滾動與 hover 動畫。」
* **產出**：AI 會在你的工作區自動建立 `index.html` 與 `index.css`。AI 還會使用瀏覽器子代理（Browser Subagent）模擬真實瀏覽器開啟網頁，為你做視覺微調、確保無版面錯誤。

## 3. 認識與使用 Skills（技能）與延伸模組

* **什麼是 Skills？**
  Skills 是 Antigravity IDE 預先封裝好的功能模組（例如文件格式轉換、特定網站抓取、格式檢查等）。
* **如何使用？**
  你**不需要**去寫代碼或下複雜命令來啟動 Skills。AI 會根據你的對話指令，在背景「自動」選擇並啟動適合張貼的 Skill 檔案（執行時會標記 `IsSkillFile: true`）。你只需要專注於用自然語言向 AI 描述你的最終目的。
* **延伸模組**：
  當需要處理特定格式（如 `.docx`、`.pptx` 或網頁）時，AI 會直接在背景呼叫系統對應的 Python 庫（如 `python-docx`、`python-pptx`）或 Node 模組。如果環境缺少該模組，AI 會自動提議安裝，你只需點擊同意即可。

## 4. Git 版本控制基本操作

IDE 在背景會自動為你的工作區維護 Git 版本庫，這讓你可以放心讓 AI 修改檔案，隨時能退回前一版本。

### 常用 Git 指令（可在 IDE 終端機執行，或直接叫 AI 幫你跑）
1. **查看變更狀態**：
   ```powershell
   git status
   ```
   *用途：看有哪些檔案被 AI 或你修改了、哪些是新建立的。*
2. **看修改細節 (Diff)**：
   ```powershell
   git diff
   ```
   *用途：逐行檢查程式碼或 Markdown 被改了什麼。*
3. **提交存檔 (Commit)**：
   ```powershell
   git add .
   git commit -m "更新行銷講稿與網頁草稿"
   ```
   *用途：將目前的修改記錄下來，建立一個安全還原點。*
4. **查看提交歷史**：
   ```powershell
   git log -n 5
   ```
   *用途：顯示最近 5 次的 commit 紀錄。*
5. **放棄修改、還原檔案**：
   ```powershell
   git checkout -- <檔案路徑>
   ```
   *用途：如果不慢意修改，用這條指令還原成上次 commit 的狀態。*

💡 **極簡秘訣**：如果你不想記指令，直接在聊天對話中打：
> 「幫我把目前的修改 commit，訊息寫：新增網頁與簡報」
AI 就會自動執行所有 Git 步驟。
