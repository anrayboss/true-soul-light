# 音影片自動轉逐字稿、字幕與大綱技能 (Transcription & Outlining Skill)

此技能讓 Agent 可以自動將本地的音影片檔案轉換成「純文字逐字稿」、「SRT 字幕檔」以及「AI 整理的結構化大綱」。

## 技能先決條件
- 系統已安裝 `ffmpeg`。
- Python 已安裝 `faster-whisper` 及 `tqdm`。
- 本地轉寫腳本位於：[tools/transcribe.py](file:///d:/Git/true-soul-light/tools/transcribe.py)

---

## Agent 執行步驟

當使用者要求對某個音訊或影片檔案（如 `.mp3`, `.wav`, `.m4a`, `.mp4`, `.mkv`, `.mov` 等）進行轉寫或整理大綱時，請依照以下步驟執行：

### 步驟 1：分析使用者要求的格式並執行轉寫腳本
根據使用者的指令，判斷需要產出的檔案格式，並傳遞 `--formats` 參數給腳本：
- **可選格式參數**：`--formats <格式列表>` （可用空格分隔多個格式，可選：`txt` 字幕文字, `srt` 字幕時間軸檔, `time` 附時間軸逐字稿）。
- **預設**：若使用者未特別指定，預設全部輸出（不需加 `--formats`，或加 `--formats txt srt time`）。
- **關鍵注意**：**由於本技能預設一律產出「大綱」，因此轉寫時必須確保輸出包含 `txt` 格式**，以便你能在後續步驟中讀取逐字稿文字。

使用 `run_command` 工具執行指令（請以目前專案根目錄執行，替換 `<檔案絕對路徑>` 及對應參數）：
```powershell
python "tools/transcribe.py" "<檔案絕對路徑>" --model small --device cpu
```
*註：可根據需求調整 `--model`、`--device` 或 `--formats`。*

### 步驟 2：讀取逐字稿內容（預設必做步驟）
使用 `view_file` 工具讀取剛剛產生的純文字逐字稿：
- 讀取路徑：`<檔案所在的資料夾>\<檔名>_逐字稿.txt`

### 步驟 3：使用 LLM 生成結構化大綱與摘要（預設必做步驟）
閱讀完逐字稿後，利用你的 LLM 分析能力，為該內容撰寫一份精美、結構化的 Markdown 大綱。

**大綱格式要求：**
- **語言**：一律使用**繁體中文**。
- **結構**：
  - `# [主題/影片標題]`
  - `## 內容核心摘要`（一句話或簡短段落說明）
  - `## 重點大綱`（使用樹狀階層 `##`, `###`, `-` 整理核心觀點與論述）
  - `## 關鍵行動清單 (Action Items)` 或 `## 重點字詞解析`（如有需要）
- **避免幻覺**：大綱內容必須完全基於逐字稿內提及的資訊，請勿捏造未提及的事實。

### 步驟 4：寫入大綱檔案（預設必做步驟）
使用 `write_to_file` 工具，將生成的大綱儲存至：
- 儲存路徑：`<檔案所在的資料夾>\<檔名>_大綱.md`

完成後，向使用者回報工作成果，並提供產出檔案的點擊連結。
