# 音影片自動轉逐字稿、字幕與大綱技能 (Transcription & Outlining Skill)

此技能讓 Agent 可以自動將本地的音影片檔案轉換成「純文字逐字稿」、「SRT 字幕檔」以及「AI 整理的結構化大綱」。

## 技能先決條件
- 系統已安裝 `ffmpeg`。
- Python 已安裝 `faster-whisper` 及 `tqdm`。
- 本地轉寫腳本位於：[tools/transcribe.py](file:///c:/Users/anray/Desktop/%E9%99%B3%E4%BB%B2%E8%B1%AA%E5%90%88%E4%BD%9C/%E5%BF%83%E6%99%BA%E5%9C%96/tools/transcribe.py)

## Agent 執行步驟

當使用者要求對某個音訊或影片檔案（如 `.mp3`, `.wav`, `.m4a`, `.mp4`, `.mkv`, `.mov` 等）進行轉寫或整理大綱時，請依照以下步驟執行：

### 步驟 1：執行轉寫腳本
使用 `run_command` 工具執行以下指令（請替換 `<檔案絕對路徑>`）：
```powershell
python "c:\Users\anray\Desktop\陳仲豪合作\心智圖\tools\transcribe.py" "<檔案絕對路徑>" --model base --device cpu
```
*註：如果使用者的電腦有安裝 NVIDIA 顯示卡且支援 CUDA，可將 `--device cpu` 改為 `--device cuda` 加速。*

執行完成後，會在**音影片檔案的同目錄下**產生三個檔案：
1. `<檔名>_逐字稿.txt`（純文字逐字稿）
2. `<檔名>_時間軸逐字稿.txt`（附帶時間標記的逐字稿）
3. `<檔名>_字幕.srt`（標準字幕檔）

### 步驟 2：讀取逐字稿內容
使用 `view_file` 工具讀取剛剛生成的純文字逐字稿：
- 讀取路徑：`<檔案所在的資料夾>\<檔名>_逐字稿.txt`
- 如果逐字稿很長，請分段讀取或全部讀取以理解其內容。

### 步驟 3：使用 LLM 生成結構化大綱與摘要
閱讀完逐字稿後，利用你的 LLM 分析能力，為該內容撰寫一份精美、結構化的 Markdown 大綱。

**大綱格式要求：**
- **語言**：一律使用**繁體中文**。
- **結構**：
  - `# [主題/影片標題]`
  - `## 內容核心摘要`（一句話或簡短段落說明）
  - `## 重點大綱`（使用樹狀階層 `##`, `###`, `-` 整理核心觀點與論述）
  - `## 關鍵行動清單 (Action Items)` 或 `## 重點字詞解析`（如有需要）
- **避免幻覺**：大綱內容必須完全基於逐字稿內提及的資訊，請勿捏造未提及的事實。

### 步驟 4：寫入大綱檔案
使用 `write_to_file` 工具，將生成的大綱儲存至：
- 儲存路徑：`<檔案所在的資料夾>\<檔名>_大綱.md`

完成後，向使用者回報工作成果，並提供這三個檔案的點擊連結。
